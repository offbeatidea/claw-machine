# 抓娃娃机 v3.3.2 产品需求文档（PRD）

> 版本：v3.3.2-build20260607e  
> 日期：2026-06-07  
> 状态：实现中  
> 作者：许清楚

---

## 1. 版本目标

在任意阶段，娃娃通过直接掉落、反弹掉落，落入出口区域时，**立即移除娃娃，并获得分数**。同时修复地面 Y 值不统一、娃娃模型与碰撞体不对齐的问题。

---

## 2. 背景与问题

### 2.1 当前问题

1. **出口判定延迟**：娃娃必须落地后才能判分，从高处落入出口区域时先反弹再得分，体验差
2. **地面 Y 值不统一**：`utils.js`、`physics.js`、`cabinet.js`、`dolls.js` 中地面 Y 值不一致，导致娃娃浮空
3. **娃娃模型与碰撞体不对齐**：娃娃模型的 Group 原点在底部，碰撞球体中心也在 Group 原点，导致碰撞球心在模型底部而非中心
4. **`dollRadius` 与 `dollSize` 无关联**：两套独立参数，导致视觉和物理对不上

---

## 3. 功能需求

### 3.1 地面 Y 值统一

**需求描述**：将所有系统中的地面 Y 值统一为 `GROUND_Y = 0.0`，确保娃娃底部刚好落在地面上。

**涉及文件**：
- `utils.js`：`GROUND_Y: 0.5` → `0.0`
- `physics.js`：移除硬编码 `groundY`，改为读 `CONFIG.GROUND_Y`
- `cabinet.js`：可见地面 `y=-0.31` → `CONFIG.GROUND_Y`
- `dolls.js`：`resetAllDolls()` 硬编码 `0.5` → `CONFIG.GROUND_Y`
- `debug-visual.js`：地面可视化默认值 `0.5` → 读 `CONFIG.GROUND_Y`

**验收标准**：
- 娃娃初始摆放后，底部刚好落在可见地面上（无浮空）
- 地面可视化圆圈与可见地面重合

---

### 3.2 娃娃初始摆放高度修正

**需求描述**：娃娃初始摆放高度公式修正为 `position.y = GROUND_Y + radius`，确保物理底部（`position.y - radius`）刚好等于 `GROUND_Y`。

**旧公式**（错误）：
```javascript
doll.position.set(x, groundY + dollSize * 0.5, z);
// 物理底部 = GROUND_Y + 0.5 - 0.3 = GROUND_Y + 0.2 ← 浮空
```

**新公式**（正确）：
```javascript
doll.position.set(x, groundY + dollRadius, z);
// 物理底部 = (GROUND_Y + radius) - radius = GROUND_Y ✓
```

**验收标准**：
- 娃娃初始摆放后，物理底部刚好等于 `GROUND_Y`
- 无浮空现象

---

### 3.3 `dollRadius` 自动跟随 `dollSize`

**需求描述**：`dollRadius`（物理碰撞半径）自动跟随 `dollSize`（视觉大小），移除独立配置项。

**计算公式**：
```javascript
dollRadius = dollSize * 0.6;
```

**涉及改动**：
1. `dolls.js`：计算 `dollRadius` 并存入 `userData`
2. `physics.js`：`registerDoll()` 从 `userData.dollRadius` 读取，移除独立默认值
3. `debug-visual.js`：从 `userData.dollRadius` 读取
4. `config.js`：移除 `dollRadius` 配置项

**验收标准**：
- 调整 `dollSizeFactor` 配置项，`dollRadius` 自动跟随变化
- 娃娃视觉大小和物理碰撞范围对齐

---

### 3.4 娃娃模型原点对齐视觉中心

**需求描述**：娃娃模型的 Group 原点从底部移到视觉中心，确保碰撞球体中心与视觉模型中心一致。

**计算方式**：
```javascript
// 身体：BoxGeometry 中心在 y=0.5*dollSize，身体底部在 y=-0.1*dollSize
// 头：SphereGeometry 中心在 y=1.4*dollSize，头部顶部在 y=1.8*dollSize
// 视觉总高：从 y=-0.1*dollSize 到 y=1.8*dollSize，中心在 y=0.85*dollSize
const visualCenterY = dollSize * 0.85;
```

**涉及改动**：
1. `dolls.js`：所有部件 Y 坐标减去 `visualCenterY`，Group 原点从底部移到中心
2. `physics.js`：`checkGroundCollision()` 和 `checkExitZone()` 改用视觉底部做判定
3. `debug-visual.js`：修正 `doll` 未定义错误

**验收标准**：
- 通过调试可视化查看，娃娃模型与碰撞球体中心对齐
- 地面碰撞检测准确（娃娃底部触地时判定为落地）

---

### 3.5 出口区域实时检测（核心新功能）

**需求描述**：娃娃在任意位置/状态，只要与地面接触点落在出口区域内，立即判分移除，不执行反弹逻辑。

**方案**：在 `checkGroundCollision()` 里，检测到地面碰撞后、执行反弹前，先判断接触点是否在出口区内。是则判分移除，不反弹；否则走原有反弹逻辑。

**判定条件**：
```javascript
// 计算娃娃底部接触点 XZ
const contactX = dollPhysics.position.x;
const contactZ = dollPhysics.position.z;
const dx = contactX - exitX;
const dz = contactZ - exitZ;
const distXZ = Math.sqrt(dx * dx + dz * dz);

if (distXZ < exitRadius) {
    // 在出口区内，立即判分移除
    scoreDollOnExitZone(dollPhysics);
    return; // 不执行反弹
} else {
    // 不在出口区内，执行原有反弹逻辑
    // ...
}
```

**优先级**：出口判定 > 地面碰撞反弹（在 `checkGroundCollision()` 内优先判断）

**验收标准**：
- 娃娃落到出口区域内时，立即得分并移除，无反弹动画
- 娃娃落到非出口区域时，正常反弹
- 娃娃从高处落入出口区域时，触地瞬间立即判分（无延迟）

---

### 3.6 得分特效

**需求描述**：娃娃得分移除时，在移除点播放得分特效。

**特效内容**：
1. **金色 `+1` 文字**：上浮 + 放大 + 淡出（持续 1.8 秒）
2. **12 个彩色粒子爆发**：向四周扩散 + 缩小 + 淡出（持续 0.9 秒）

**涉及改动**：
- `claw.js` 新增 `playScoreEffect(position)` 函数

**验收标准**：
- 娃娃得分移除时，移除点播放金色 `+1` 文字和彩色粒子
- 特效持续时间内完成动画，然后自动移除 DOM 元素

---

### 3.7 移除动画（闪烁渐隐）

**需求描述**：娃娃被判分移除时，先播放闪烁渐隐动画，动画完成后再判分+移除。

**动画流程**：
```
checkExitZone() 判定命中
    ↓
调用 Claw.scoreDollOnExitZone(dollPhysics)
    ↓
标记 dollPhysics._removing = true（physics.js 跳过此娃娃的物理更新）
    ↓
播放闪烁渐隐动画（0.5秒）
    ↓
动画完成 → 判分 + 移除娃娃 + 播放得分特效
```

**闪烁渐隐动画细节**：
- 时长：500ms
- 闪烁：opacity 在 `1.0 ↔ 0.3` 之间快速切换（每 100ms 一次）
- 渐隐：最后 200ms opacity 从当前值降到 0
- 动画完成后回调：判分、移除娃娃、播得分特效

**涉及改动**：
- `claw.js` 修改 `scoreDollOnExitZone()` 函数，加入闪烁渐隐动画逻辑
- `physics.js` 修改 `update()` 函数，循环开头增加 `if (dollPhysics._removing) return;`

**验收标准**：
- 娃娃被判分移除时，先播放闪烁渐隐动画
- 动画完成后，娃娃被移除，得分 +1，播放得分特效
- 动画过程中，娃娃不受物理引擎影响（不移动、不旋转）

## 4. 非功能需求

### 4.1 性能要求
- 出口区域实时检测不影响帧率（每帧检测次数 = 娃娃数量）
- 得分特效使用 CSS 动画，不阻塞主线程

### 4.2 兼容性要求
- 支持 Chrome、Edge 最新版本
- 支持 Windows 10/11

## 6. 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 出口判定误触 | 娃娃在出口区上方但未落地时被误判 | 只在地面碰撞时判定，不提前判分 |
| 闪烁渐隐动画卡顿 | 低端设备动画不流畅 | 使用 CSS 动画，减少 JS 计算 |
| 得分特效 DOM 泄漏 | 特效元素未正确移除 | 使用 `setTimeout` 确保移除 |
