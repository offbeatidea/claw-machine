## 二、模块职责划分

### 2.1 `code/js/config.js` - 配置面板
**职责**:
- 配置面板UI创建/显示/隐藏
- 配置参数读取/写入
- 配置变更事件监听
- 参数解释文字

**依赖**:
- `window.currentConfig` 对象（在`main.js`中定义）
- `window.CONFIG` 常量（在`utils.js`中定义）

**导出**:
```javascript
window.ConfigManager = {
    init(),           // 初始化配置面板
    getConfig(),      // 获取当前配置
    updateConfig()    // 更新配置
};
```

---

### 2.2 `code/js/physics.js` - 物理引擎
**职责**:
- Cannon.js世界创建/管理
- 刚体创建（娃娃/地面/机箱墙壁）
- 碰撞检测/响应
- 物理参数配置（重力/阻尼/反弹等）

**依赖**:
- `CANNON` 全局对象（通过`<script>`引入）
- `window.currentConfig` 对象（获取物理参数）

**导出**:
```javascript
window.PhysicsManager = {
    init(),                // 初始化物理世界
    createDollBody(doll), // 为娃娃创建刚体
    createGroundBody(),    // 创建地面刚体
    createWallBodies(),   // 创建机箱墙壁
    step(world, deltaTime), // 推进物理世界
    syncBodiesToMeshes()  // 同步Cannon.js → Three.js
};
```

## 四、关键技术点

### 4.1 全局变量管理
**问题**: 拆分后，模块间需要共享变量（如`scene`, `camera`, `dolls`等）

**解决方案**: 使用`window`对象挂载全局变量
```javascript
// main.js
window.scene = new THREE.Scene();
window.camera = new THREE.PerspectiveCamera(...);
window.renderer = new THREE.WebGLRenderer(...);
window.dolls = [];
window.currentConfig = {...};

// 其他模块可以直接访问
// window.scene, window.dolls, window.currentConfig
```

---

### 4.2 模块加载顺序
**问题**: JS文件加载顺序很重要（依赖关系）

**解决方案**: 在`index.html`中按依赖顺序添加`<script>`标签
```html
<!-- 1. 第三方库 -->
<script src="code/libs/cannon.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js"></script>

<!-- 2. 工具模块（无依赖） -->
<script src="code/js/utils.js"></script>

<!-- 3. 功能模块（按依赖顺序） -->
<script src="code/js/config.js"></script>
<script src="code/js/physics.js"></script>
<script src="code/js/cabinet.js"></script>
<script src="code/js/claw.js"></script>
<script src="code/js/dolls.js"></script>

<!-- 4. 主入口（依赖所有模块） -->
<script src="code/js/main.js"></script>
```

---

### 4.3 Cannon.js与Three.js坐标同步
**问题**: Cannon.js刚体位置需要同步到Three.js网格

**解决方案**: 在`animate()`主循环中每帧同步
```javascript
function animate() {
    requestAnimationFrame(animate);
    
    // 更新物理世界
    PhysicsManager.step(world, 1/60);
    
    // 同步娃娃位置（Cannon.js → Three.js）
    dolls.forEach((doll, index) => {
        const body = doll.userData.physicsBody;
        if (body) {
            doll.position.copy(body.position);
            doll.quaternion.copy(body.quaternion);
        }
    });
    
    renderer.render(scene, camera);
}
```

### 4.4 Cannon.js基础用法（参考）

```javascript
// 1. 创建物理世界
const world = new CANNON.World();
world.gravity.set(0, -9.8, 0); // 重力
world.broadphase = new CANNON.NaiveBroadphase();
world.solver.iterations = 10;

// 2. 创建地面刚体
const groundBody = new CANNON.Body({
    mass: 0, // 质量=0 = 静态物体
    shape: new CANNON.Plane()
});
groundBody.quaternion.setFromAxisAngle(new CANNON.Vec3(1, 0, 0), -Math.PI / 2);
world.addBody(groundBody);

// 3. 创建娃娃刚体（球体或盒子）
const dollBody = new CANNON.Body({
    mass: 1, // 质量>0 = 动态物体
    shape: new CANNON.Sphere(0.3)
});
dollBody.position.set(x, y, z);
world.addBody(dollBody);

// 4. 碰撞事件监听
dollBody.addEventListener('collide', (event) => {
    console.log('娃娃碰撞了！', event);
});

// 5. 每帧推进物理世界
world.step(1/60); // 时间步长
```
