# 抓取流程与状态标记说明

> 文档版本：v1.0 | 更新日期：2026-06-08
> 目的：完整描述抓娃娃机一次抓取流程中各阶段、状态标记、数组引用的变化，用于排查"娃娃不跟随爪子"类问题。

---

## 一、各个娃娃列表的含义

| 列表 | 挂载位置 | 类型 | 含义 | 生命周期 |
|------|---------|------|------|---------|
| `DollManager.dolls` | DollManager 实例 | Array\<mesh\> | 全局娃娃数组，机箱内所有娃娃 | 游戏开始创建 → 得分后移除 |
| `Claw.grabbedDolls` | Claw 实例 | Array\<mesh\> | 爪子当前抓到的娃娃（强抓+弱抓判定前都在里面） | `onCollisionDetected()` 成功时添加 → `releaseAllDolls()` 或 `judgeStrongGrab()` 时清空 |
| `Claw.weakGrabDolls` | Claw 实例 | Array\<mesh\> | 弱抓判定后掉落的娃娃 | `judgeStrongGrab()` 时添加 → 掉落物理模拟完成后移除 |
| `Claw.releasedDolls` | Claw 实例 | Array\<mesh\> | 爪子松开后等待落地判分的娃娃 | `releaseAllDolls()` 时从 `grabbedDolls` 复制 → `scoreDollOnLanding()` 或 `finishRemoveDoll()` 时移除 |

---

## 二、完整流程中的列表变化

### ① 初始状态

```
DollManager.dolls = [小熊(id=0), 兔子(id=1), 猫咪(id=2), 狗狗(id=3), 企鹅(id=4)]
Claw.grabbedDolls  = []
Claw.weakGrabDolls = []
Claw.releasedDolls = []
```

---

### ② 抓取检测（`onCollisionDetected()` 成功）

```
Claw.grabbedDolls.push(doll)     ← 娃娃被加入抓取列表
doll.userData.isGrabbed = true    ← 标记已抓取
物理引擎: physObj.state = 'grabbed'
```

**示例**（抓到"小熊"）：
```
DollManager.dolls = [小熊(id=0), 兔子(id=1), 猫咪(id=2), 狗狗(id=3), 企鹅(id=4)]
Claw.grabbedDolls  = [小熊(id=0)]
Claw.weakGrabDolls = []
Claw.releasedDolls = []
```

---

### ③ 上升过程（`updateAscending()`）

- `grabbedDolls` 中的娃娃跟随爪子移动（在 `update()` 中处理）
- 爪子动画闭合（`animateClawClose()`）

**列表无变化**，只是位置跟随。

---

### ④ 强抓/弱抓判定（`judgeStrongGrab()`）

#### 强抓（isStrong = true）

```
strongDolls = grabbedDolls（所有娃娃）
weakDolls = []
Claw.grabbedDolls = strongDolls   ← 保留强抓娃娃
Claw.weakGrabDolls = []            ← 空
```

**示例**（强抓"小熊"）：
```
DollManager.dolls = [小熊(id=0), 兔子(id=1), 猫咪(id=2), 狗狗(id=3), 企鹅(id=4)]
Claw.grabbedDolls  = [小熊(id=0)]
Claw.weakGrabDolls = []
Claw.releasedDolls = []
```

#### 弱抓（isStrong = false）

```
strongDolls = []
weakDolls = grabbedDolls（所有娃娃）
Claw.grabbedDolls = []             ← 清空！
Claw.weakGrabDolls = weakDolls    ← 弱抓娃娃掉落
调用 applyParabolicDropWithVelocity() 赋予初速度
```

**示例**（弱抓"小熊"）：
```
DollManager.dolls = [小熊(id=0), 兔子(id=1), 猫咪(id=2), 狗狗(id=3), 企鹅(id=4)]
Claw.grabbedDolls  = []
Claw.weakGrabDolls = [小熊(id=0)]
Claw.releasedDolls = []
```

---

### ⑤ 返回出口（`updateReturning()` → `releaseAllDolls()`）

**强抓娃娃到达出口上方时**：

```
animateClawOpen()               ← 播放张开动画
releaseAllDolls() 被调用：
  Claw.releasedDolls = Claw.grabbedDolls.slice()  ← 复制到 releasedDolls
  Claw.grabbedDolls = []                      ← 清空
  娃娃物理状态改为 'falling'
进入 'releasing' 状态，等待娃娃落地
```

**示例**（强抓"小熊"，到达出口上方）：
```
DollManager.dolls = [小熊(id=0), 兔子(id=1), 猫咪(id=2), 狗狗(id=3), 企鹅(id=4)]
Claw.grabbedDolls  = []
Claw.weakGrabDolls = []
Claw.releasedDolls = [小熊(id=0)]
```

---

### ⑥ 落地判分（`scoreDollOnLanding()` 或 `finishRemoveDoll()`）

#### 娃娃在出口区域内（得分）

```
window.gameScore++             ← 得分
从 DollManager.dolls 中移除娃娃
从 PhysicsEngine.dolls 中移除物理对象
从 Claw.releasedDolls 中移除娃娃
如果 releasedDolls 已空，进入 'idle' 状态
```

**示例**（"小熊"得分）：
```
DollManager.dolls = [兔子(id=1), 猫咪(id=2), 狗狗(id=3), 企鹅(id=4)]
Claw.grabbedDolls  = []
Claw.weakGrabDolls = []
Claw.releasedDolls = []
```

#### 娃娃不在出口区域内（未得分）

```
娃娃留在机箱内，继续物理模拟
从 Claw.releasedDolls 中移除娃娃（已判分）
```

**示例**（"小熊"未得分）：
```
DollManager.dolls = [小熊(id=0), 兔子(id=1), 猫咪(id=2), 狗狗(id=3), 企鹅(id=4)]
Claw.grabbedDolls  = []
Claw.weakGrabDolls = []
Claw.releasedDolls = []
```

---

### ⑦ 弱抓娃娃落地（不在 `releasedDolls` 中）

弱抓娃娃直接由物理引擎模拟，不在 `releasedDolls` 中，所以落地时不会调用 `scoreDollOnLanding()`。

如果弱抓娃娃落入出口区域，需要在 `checkExitZone()` 中检测（**注意**：当前版本已删除 `checkExitZone()` 的不限高度检测，只在落地时判分）。

---

## 三、涉及的状态标记总览

| 标记 | 挂载位置 | 类型 | 作用 |
|------|---------|------|------|
| `doll.userData.isGrabbed` | doll mesh | Boolean | 娃娃是否被爪子抓取中 |
| `doll.userData.grabAttempted` | doll mesh | Boolean | 本轮抓取是否已参与过判定（防重复掷骰子）|
| `doll.userData.grabOffset` | doll mesh | {x,y,z} | 爪子与娃娃的偏移量（跟随用）|
| `claw.grabbedDolls[]` | Claw 实例 | Array\<mesh\> | 当前抓到的娃娃列表（强抓+弱抓判定前都在里面）|
| `claw.weakGrabDolls[]` | Claw 实例 | Array\<mesh\> | 弱抓判定后待掉落的娃娃 |
| `claw.releasedDolls[]` | Claw 实例 | Array\<mesh\> | 已释放到出口、等待落地判分的娃娃 |
| `dollPhysics.state` | physics | String | 娃娃物理状态（falling/bouncing/rolling/resting）|
| `dollPhysics._removed` | physics | Boolean | 娃娃是否已从物理引擎移除 |
| `window.gameState` | 全局 | String | 游戏状态机（idle/descending/grabbing/ascending/ascend_done/returning/releasing/scoring）|

## 二、完整抓取流程（时序）

### ① `grab()` — 玩家按下抓取按钮

**状态变化**：`gameState: idle → descending`

**标记操作**：
- 清除所有娃娃的 `doll.userData.grabAttempted = false`
- 重置爪子位置（`resetPendulumState()`）
- 重置绳长 `currentRopeLength = fullRope`

**涉及数组**：无变化

---

### ② `updateDescending()` — 爪子下降过程

**状态**：`descending`

**每帧检测**：
- `checkCollision()` 被调用（见③）

**标记操作**：无（碰撞检测在③处理）

---

### ③ `checkCollision()` — 下降中碰撞检测（每帧）

**遍历对象**：`window.DollManager.dolls`（mesh 数组）

**命中条件**：
- `!doll.userData.isGrabbed`（未被抓取）
- `!doll.userData.grabAttempted`（本轮未判定过）
- 距离 < `CLAW_COLLISION_RADIUS`

**命中后**：
- 标记 `doll.userData.grabAttempted = true`
- 调用 `onCollisionDetected(doll)`

**重要**：每个娃娃每轮只判定一次，用 `grabAttempted` 保证。

---

### ④ `onCollisionDetected(doll)` — 单次命中处理

**掷骰子**：`Math.random() < grabSuccessRate`

**成功时**：
- `claw.grabbedDolls.push(doll)`
- `doll.userData.isGrabbed = true`
- 计算 `doll.userData.grabOffset = {x, y, z}`（爪子世界坐标 - 娃娃世界坐标）
- 显示浮动文字："抓到【XX】"

**失败时**：
- 播放抓取失败动画（`animateGrabFail`）
- 娃娃继续物理模拟（不被爪子影响）

---

### ⑤ `updateDescending()` — 继续下降直到触底

**触发条件**：`currentRopeLength <= 0`（或达到最大深度）

**状态变化**：`gameState: descending → ascend_done`（中间调用 `onGrabComplete()`）

---

### ⑥ `onGrabComplete()` — 触底处理

**状态变化**：`gameState: ascend_done → ascending`

**标记操作**：
- 播放爪子闭合动画 `animateClawClose()`
- 此时 `grabbedDolls[]` 中已有所有成功抓取的娃娃

---

### ⑦ `updateAscending()` — 爪子上升过程

**状态**：`ascending`

**每帧操作**：
- 缩短绳长 `currentRopeLength -= ascendSpeed * dt`
- **关键**：`grabbedDolls[]` 中的每个娃娃跟随爪子移动：
  ```js
  swingWorldPos = this.getSwingWorldPos();
  for (const doll of this.grabbedDolls) {
      doll.position.x = swingWorldPos.x + doll.userData.grabOffset.x;
      doll.position.y = swingWorldPos.y + doll.userData.grabOffset.y;
      doll.position.z = swingWorldPos.z + doll.userData.grabOffset.z;
  }
  ```

**触发条件**：`currentRopeLength <= target`（上升到顶）

**状态变化**：`gameState: ascending → ascend_done`（中间调用 `judgeStrongGrab()`）

---

### ⑧ `judgeStrongGrab()` — 强抓/弱抓判定（爪子级别，一次掷骰子）

> **重要**：强抓/弱抓是**爪子级别**判定，不是每个娃娃单独判定。
> 一次抓取只掷一次骰子，所有抓到的娃娃统一为强抓或弱抓。

**掷骰子**：`isStrong = Math.random() < strongGrabRate`

**强抓时**（`isStrong = true`）：
- `claw.grabbedDolls` 保留所有娃娃（不转移）
- 所有娃娃标记为强抓

**弱抓时**（`isStrong = false`）：
- 所有 `grabbedDolls` 中的娃娃移入 `claw.weakGrabDolls`
- `claw.grabbedDolls = []`（清空）
- 调用 `applyParabolicDropWithVelocity()` 给弱抓娃娃赋予抛物线掉落初速度

**状态变化**：`gameState: ascend_done → returning`

---

### ⑨ `updateReturning()` — 爪子返回出口上方

**状态**：`returning`

**每帧操作**：
- 移动爪子底座到出口位置
- 缩短绳长到目标值

**触发条件**：爪子到达出口上方（`|base.x - exitX| < threshold && |base.z - exitZ| < threshold`）

**操作**：
- 播放爪子张开动画 `animateClawOpen()`
- 调用 `releaseAllDolls()`

---

### ⑩ `releaseAllDolls()` — 释放强抓娃娃到出口

**遍历对象**：`claw.grabbedDolls`（强抓娃娃）

**操作**：
- 将娃娃位置设置为出口上方
- 标记 `doll.userData.isGrabbed = false`
- 将娃娃移入 `claw.releasedDolls[]`
- 调用 `finishRemoveDoll(doll)` 从各系统移除

**状态变化**：`gameState: returning → releasing`

---

### ⑪ `finishRemoveDoll(doll)` — 从所有系统移除娃娃

**操作**：
1. 从 `claw.releasedDolls` 移除（`indexOf + splice`）
2. 从 `claw.grabbedDolls` 移除（`indexOf + splice`）
3. 从 `DollManager.dolls` 移除（`splice`）
4. 从物理引擎移除（`PhysicsEngine.removeDoll`）
5. 从场景中移除 mesh（`scene.remove(doll)`）

**重要**：步骤2用 `indexOf(doll)` 查找，要求传入的 `doll` 引用必须与 `grabbedDolls[]` 中的是同一个对象。

---

### ⑫ `checkGroundCollision()` — 娃娃落地检测（物理引擎每帧）

**遍历对象**：`PhysicsEngine.dolls`（physics 数组）

**命中条件**：`dollPhysics.position.y - radius <= GROUND_Y`

**落地后**：
- 标记 `dollPhysics.state = 'resting'`
- 调用 `scoreDollOnLanding(dollPhysics)`

---

### ⑬ `scoreDollOnLanding(dollPhysics)` — 落地判分

**操作**：
- 判断是否在出口区域内
- 是：得分 +1，播放得分特效
- 调用 `DollManager.finishRemoveDoll(dollPhysics)`（注意：这里传入的是 `dollPhysics`，不是 mesh）

---

## 三、问题排查关键点

### 问题表现：抓到娃娃后，娃娃不跟随爪子移动

**可能根因**：

1. **`grabbedDolls[]` 中存的对象引用错误**
   - `onCollisionDetected()` 传入的 `doll` 是 mesh
   - `grabbedDolls.push(doll)` 存的是 mesh 引用
   - `updateAscending()` 中遍历 `grabbedDolls` 时用的是同一个引用
   - **如果 `DollManager.dolls` 在中间被修改（如娃娃被移除后数组 splice），但 `grabbedDolls[]` 中的引用还是有效的（因为引用指向对象本身，不是数组索引）**

2. **`grabOffset` 计算错误**
   - `onCollisionDetected()` 中计算偏移时，爪子位置可能还没更新
   - 或者娃娃的 world position 计算错误

3. **`updateAscending()` 中的跟随逻辑没有执行**
   - `grabbedDolls.length === 0`（娃娃被错误地提前移除）
   - 或者 `gameState` 不是 `ascending`

4. **引用错乱：多个娃娃对象混淆**
   - `DollManager.dolls` 是 mesh 数组，每个 mesh 有唯一的 `userData.id`
   - 如果 `onCollisionDetected(doll)` 传入的 `doll` 不是预期的那个（比如数组索引错乱），那么 `grabbedDolls` 中存的就是错误的娃娃

## 四、诊断日志方案

在以下时机输出 `grabbedDolls`、`releasedDolls`、`DollManager.dolls` 的全列表：

| 时机 | 位置 | 输出内容 |
|------|------|---------|
| 【按下抓取按钮】 | `grab()` 开头 | `DollManager.dolls` 全列表（name + id）|
| 【抓到判断】 | `onCollisionDetected()` 开头 | 传入的 `doll`（name + id）+ `grabbedDolls` 全列表 |
| 【强弱抓判断前】 | `judgeStrongGrab()` 开头 | `grabbedDolls` 全列表（name + id）|
| 【强弱抓判断后】 | `judgeStrongGrab()` 结尾 | `grabbedDolls` 全列表 + `weakGrabDolls` 全列表 |
| 【出口顶端松爪判断前】 | `updateReturning()` 到达出口上方时 | `grabbedDolls` 全列表 |
| 【出口顶端松爪判断后】 | `releaseAllDolls()` 开头 | `grabbedDolls` 全列表 |
| 【娃娃落地】 | `checkGroundCollision()` 落地时 | `dollPhysics` 对应的 mesh（通过 id 查找）+ `releasedDolls` 全列表 |
| 【得分前】 | `scoreDollOnLanding()` 开头 | `dollPhysics` 信息 + `releasedDolls` 全列表 |
| 【得分后】 | `scoreDollOnLanding()` 结尾 | 移除后的 `releasedDolls` 全列表 |
| 【finishRemoveDoll】 | `finishRemoveDoll()` 开头 | 传入的 `doll`（name + id）+ `grabbedDolls` 全列表 + `releasedDolls` 全列表 + `DollManager.dolls` 全列表 |
