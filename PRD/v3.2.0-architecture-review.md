## 一、当前架构概述

### 1.1 技术栈
- **渲染引擎**: Three.js（3D渲染）
- **物理系统**: ❌ **无**（当前使用线性动画）
- **状态管理**: 简单状态枚举（`GameState`）
- **代码结构**: 单文件HTML（~2200行），所有逻辑在`init()`函数中

### 1.2 核心数据结构

#### 娃娃对象（THREE.Group）
```javascript
doll.userData = {
    id: i,
    name: names[i],
    color: colors[i],
    isGrabbed: false
    // ❌ 缺少：velocity, state, bounceCount, rotationSpeed
};
```

**问题**: 没有物理属性（速度、状态、反弹计数等）

#### 爪子对象
- `clawBase`: 底座组（玩家控制移动）
- `clawSwing`: 摆动组（钟摆物理）
- `clawFingers`: 3个爪指数组

**状态管理**:
```javascript
GameState = { IDLE, DESCENDING, GRABBING, ASCENDING, MOVING_TO_EXIT }
this.currentState = GameState.IDLE;
this.grabbedDolls = []; // 当前抓取的娃娃列表
```

### 1.3 当前掉落逻辑（第1710-1725行）

**❌ 严重问题：线性掉落，非抛物线**

```javascript
animateDollDrop(doll) {
    const groundY = 0.5;
    const dropSpeed = 0.05; // 固定速度
    
    const drop = () => {
        if (doll.position.y > groundY) {
            doll.position.y -= dropSpeed; // ❌ 匀速下降
            requestAnimationFrame(drop);
        } else {
            doll.position.y = groundY;
            doll.userData.isGrabbed = false;
        }
    };
    drop();
}
```

**缺陷**:
1. 无重力模拟（应该是加速度，不是匀速）
2. 无空气阻力
3. 无水平速度（X/Z固定）
4. 无反弹
5. 使用`requestAnimationFrame`递归，不在主循环中统一管理

## 三、架构重构建议

### 3.1 总体方案

**推荐方案**: 在现有代码基础上添加**轻量级物理引擎**（不引入外部库，自己实现）

**理由**:
1. 项目是单文件HTML，引入外部物理库（如Cannon.js、Ammo.js）会大幅增加复杂度
2. 抓娃娃机物理需求相对简单（抛物线、碰撞、反弹），自己实现更可控
3. 可以复用现有的Three.js物体属性（position、rotation）

### 3.2 数据结构扩展

#### 扩展娃娃对象


### 3.3 物理引擎实现

#### 方案A：集成到现有主循环（推荐）

找到游戏的`animate()`主循环函数（应该是使用`requestAnimationFrame`递归调用的），在其中添加物理更新：

```javascript
function animate() {
    requestAnimationFrame(animate);
    
    const deltaTime = clock.getDelta();

    updateClawPosition(deltaTime);
```

#### 物理更新函数

```javascript
function updateDollPhysics(doll, deltaTime) {
    const ud = doll.userData;
    
    // 跳过不需要物理更新的状态
    if (ud.state === DollState.RESTING || ud.state === DollState.GRABBED) {
        return;
    }
    
    // 1. 应用重力
    ud.velocity.y -= currentConfig.gravity * deltaTime;
    
    // 2. 应用空气阻力
    const airResistance = currentConfig.airResistance || 0.02;
    ud.velocity.x *= (1 - airResistance);
    ud.velocity.z *= (1 - airResistance);
    
    // 3. 更新位置
    doll.position.x += ud.velocity.x * deltaTime;
    doll.position.y += ud.velocity.y * deltaTime;
    doll.position.z += ud.velocity.z * deltaTime;
    
    // 4. 更新旋转（翻滚效果）
    doll.rotation.x += ud.rotationSpeed.x * deltaTime;
    doll.rotation.z += ud.rotationSpeed.z * deltaTime;
    
    // 5. 边界检测（限制在机箱内）
    constrainDollToCabinet(doll);
    
    // 6. 娃娃间碰撞检测
    if (ud.state === DollState.FALLING || ud.state === DollState.BOUNCING) {
        checkDollCollisions(doll);
    }
    
    // 7. 落地/落娃检测
    checkDollLanding(doll, deltaTime);
}
```

### 3.4 关键函数实现

#### 落地检测函数

```javascript
function checkDollLanding(doll, deltaTime) {
    const ud = doll.userData;
    
    // 计算地面高度（可能是地面或其他娃娃顶部）
    let groundY = 0; // 地面高度
    
    // 检查是否落在其他娃娃上
    for (const otherDoll of dolls) {
        if (otherDoll === doll) continue;
        if (otherDoll.userData.state === DollState.RESTING || 
            otherDoll.userData.state === DollState.SETTLED) {
            
            // 简单的X-Z平面距离检测
            const dx = doll.position.x - otherDoll.position.x;
            const dz = doll.position.z - otherDoll.position.z;
            const distance = Math.sqrt(dx*dx + dz*dz);
            
            if (distance < DOLL_RADIUS * 2) {
                // 落在其他娃娃上
                const otherTop = otherDoll.position.y + DOLL_HEIGHT;
                if (otherTop > groundY) {
                    groundY = otherTop;
                }
            }
        }
    }
    
    // 检测是否触地
    if (doll.position.y <= groundY + DOLL_HEIGHT / 2) {
        // 触地
        doll.position.y = groundY + DOLL_HEIGHT / 2;
        
        // 反弹逻辑
        ud.bounceCount++;
        const bounceDamping = currentConfig.bounceDamping || 0.6;
        ud.velocity.y = -ud.velocity.y * bounceDamping;
        
        // 摩擦力
        const friction = currentConfig.friction || 0.9;
        ud.velocity.x *= friction;
        ud.velocity.z *= friction;
        
        // 状态转换
        if (ud.bounceCount >= (currentConfig.maxBounces || 3) ||
            Math.abs(ud.velocity.y) < 0.01) {
            ud.state = DollState.SETTLED;
            ud.velocity.set(0, 0, 0);
            console.log(`[Physics] 娃娃"${ud.name}"已静止`);
        } else {
            ud.state = DollState.BOUNCING;
        }
    }
}
```

## 六、结论与建议

### 6.1 结论

当前架构**不支持**v3.2.0需求，需要**重构掉落系统**并**添加轻量级物理引擎**。

**核心问题**:
1. 掉落是线性的（非抛物线）
2. 无物理属性（速度、状态等）
3. 无碰撞检测
4. 无反弹系统

### 6.2 建议

**✅ 推荐方案**: 在现有代码基础上自己实现物理引擎（不引入外部库）

**理由**:
1. 项目是单文件HTML，引入外部库会增加复杂度
2. 抓娃娃机物理需求相对简单，自己实现更可控
3. 可以复用现有的Three.js对象

**⚠️ 注意事项**:
1. **增量开发**：每完成一个Phase就测试，避免一次性大改
2. **性能监控**：使用浏览器FPS meter，确保性能不下降
3. **代码备份**：每次大改前备份`index.html`
