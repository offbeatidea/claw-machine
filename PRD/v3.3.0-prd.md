### 2.2 抓取系统（重大重构 v330i~v330j）

#### 2.2.1 多娃娃抓取

**旧实现**：`grabbedDoll`（单数，一次只抓一个）
**新实现**：`grabbedDolls`（数组，支持同时抓多个）

涉及修改的 9 处函数：

| # | 函数 | 修改内容 |
|---|------|----------|
| 1 | `update()` | 遍历 `grabbedDolls` 数组，每个娃娃用各自的 `grabOffset` 跟随 |
| 2 | `checkCollision()` | 移除 `hasGrabbedOne` 限制 |
| 3 | `onGrabComplete()` | 日志显示所有娃娃名 |
| 4 | `judgeStrongGrab()` | 逐个判定强抓/弱抓，强抓留在 `grabbedDolls`，弱抓移到 `weakGrabDolls` |
| 5 | `applyParabolicDrop*()` | 遍历 `weakGrabDolls` 数组 |
| 6 | 到达出口检查 | `grabbedDolls.length > 0` → 调用 `releaseAllDolls()` |
| 7 | `releaseAllDolls()` | 新函数，遍历释放所有娃娃，保存到 `releasedDolls` |
| 8 | `judgeScore()` | 只检查 `releasedDolls` 而非扫描全部娃娃 |
| 9 | `resetPendulumState()` | 清理 `grabbedDolls`/`weakGrabDolls`/`releasedDolls` |

#### 2.2.2 释放→自由落体→判分流程（全新）

**旧实现**：强抓到达出口后直接传送到出口位置，瞬间判定得分
**新实现**：完整物理流程

```
强抓到达出口上方
    ↓
releaseAllDolls()
  - 张开爪子（animateClawOpen）
  - 娃娃获得随机水平初速度
  - 物理引擎接管（state='falling'）
  - 保存到 releasedDolls[]
    ↓
进入 releasing 状态
    ↓
checkDollsLanded()
  - 等待所有 releasedDolls:
    physObj.state === 'resting'
    && physObj.onGround === true
    ↓
全部落地 → 进入 scoring 状态
    ↓
judgeScore()
  - 遍历 releasedDolls
  - 检查与出口距离 < exitRadius
  - 命中：得分 + 从 scene/DollManager/PhysicsEngine 删除模型
  - 未命中：留在机箱中
    ↓
回到 idle 状态
```

### 2.3 浮动文字提示系统

| 事件 | 提示内容 | 触发函数 |
|------|----------|----------|
| 抓到娃娃 | "抓到【娃娃名1, 娃娃名2】"（多个全列出） | `onCollisionDetected()` |
| 强抓判定 | "强抓！" | `judgeStrongGrab()` |
| 弱抓判定 | "没抓住" / "掉了" / "抓薄了"（3选1随机） | `judgeStrongGrab()` |
| 松开掉落 | "松开！娃娃名 掉落中..." | `releaseAllDolls()` |
| 得分 | "得分 +N！" | `judgeScore()` |

**技术实现**：DOM 元素 + `window.gameCamera.project()` 坐标投影 + 自动消失

### 2.4 抓取流程完整状态机

```
idle → descending → grabbing → ascending → returning → releasing → scoring → idle
```

| 状态 | 入口条件 | 出口条件 | 关键行为 |
|------|----------|----------|----------|
| idle | 上一轮结束 / 初始 | 玩家按抓取键 | 接收输入，更新 clawVelocity |
| descending | 按抓取 | 爪子到达底部 | 绳子伸长，爪子下降 |
| grabbing | 到达底部 | 碰撞检测完成 | 闭合爪子，判定抓取结果 |
| ascending | 抓取判定完成 | 上升到目标高度 | 收绳，强抓/弱抓分叉处理 |
| returning | 上升完成 | 到达出口上方 | 移动到出口 XZ 位置 |
| releasing | 到达出口 | 所有娃娃落地静止 | 松开爪子，娃娃自由掉落（**新增 v330j**） |
| scoring | 全部娃娃落地 | 判定完成 | 检查出口区域，计分，删除模型 |

### 2.5 版本管理系统（新增 v330k）

#### 文件结构

```
claw-machine/
├── version.json          # 单一版本信息源
├── CHANGELOG.md          # 版本历史文档
├── update-version.js     # Node.js 版本同步脚本
├── update-version.bat   # Windows 批处理工具
└── .gitignore            # Git 忽略规则
```

#### 版本号格式

```
v{major}.{minor}.{patch}-build{YYYYMMDD}{a-z}
```

示例：`v3.3.0-build20260530j`

- `major.minor.patch`：语义化版本号
- `build`：日期 + 随机字母（用于缓存清除和唯一标识）

#### 更新命令

```bash
node update-version.js          # patch +1
node update-version.js --minor   # minor +1
node update-version.js --major   # major +1
node update-version.js --build   # 仅更新时间戳
```

### 5.2 核心数据结构

#### 娃娃对象
```javascript
doll.userData = {
    id: i,
    name: names[i],
    isGrabbed: false,
    grabOffset: { x, y, z }  // 相对爪子的偏移量
};
```

#### 物理对象（PhysicsEngine）
```javascript
physObj = {
    mesh: doll,
    position: new THREE.Vector3(),
    velocity: new THREE.Vector3(),
    rotationSpeed: new THREE.Vector3(),
    state: 'grabbed' | 'falling' | 'resting',
    onGround: false,
    bounceCount: 0
};
```

#### 爪子系统（Claw）
```javascript
Claw = {
    group, base, swingGroup, rope, fingers[3],
    grabbedDolls: [],       // 强抓保持的娃娃
    weakGrabDolls: [],      // 弱抓掉落的娃娃
    releasedDolls: [],      // 释放后等待判分的娃娃
    clawVelocity: { x: 0, z: 0 },  // 基座速度（惯性力计算用）
    pendulumAngleX/Z,       // X/Z 轴摆角
    pendulumAngVelX/Z,      // 角速度
    currentRopeLength,      // 当前绳长
    ropeAnchorY             // 绳子挂点Y坐标
};
```

## 八、开发规范

### 8.1 文件编辑规范

| 规则 | 说明 | 原因 |
|------|------|------|
| 禁止 Node.js 字符串替换 | 不使用 `-e` 脚本做 sed/replace | CRLF 换行符 + 模板字符串反引号导致匹配失败和文件截断 |
| 使用 Write 工具 | 修改文件时写完整文件内容 | 避免 CRLF/编码问题 |
| 备份优先 | 重大修改前先备份 | 无 Git 时唯一的恢复手段（现已接入 Git） |

### 8.2 Git 提交规范

```
v{version}: {简洁变更描述}

- 详细变更1
- 详细变更2
```

示例：
```
v3.3.0-build20260530j: 初始版本提交

- Three.js 3D抓娃娃机游戏
- 文件拆分架构
- 钟摆物理机制
- 多娃娃抓取支持
```

### 8.3 版本更新流程

```bash
# 1. 更新版本号
update-version.bat  # 或 node update-version.js

# 2. 编辑 version.json 的 changes 字段（记录变更）

```
