## 新需求评估流程

当提出新需求时，按以下顺序评估：

1. **查重**：在本表中搜索是否已有相同或相似需求
2. **合并**：若与某待开发项高度重叠 → 合并到该条目，不新增
3. **优先级**：是否属于 Phase 2~3 核心物理/抓取逻辑？是 → 优先；否 → 考虑推迟
4. **依赖检查**：是否有前置任务未完成？有 → 建议先完成前置
5. **范围检查**：是否与五线并行其他项目冲突？是否超出抓娃娃机范围？

**建议推迟的情况：**
- 属于 Phase 5（玩法扩展）且 Phase 2~3 未完成
- 纯美术/音效需求（不影响核心玩法）
- 与当前迭代目标无关的边缘功能
